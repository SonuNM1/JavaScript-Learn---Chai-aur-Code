-------- var ----------

1. function scope / global scope 
2. hoited to the top of their function 
3. variable declared with 'var' is accessible outside the block
4. can be redeclared in the same scope and won't throw an error 


------- let --------

1. block scope 
2. not accessible outside the block 
3. cannot redeclared in the same block, will throw an error 


thus, let recommended
